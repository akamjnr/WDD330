Introduction to Functions, Execution Context and the Call Stack

When JavaScript code runs, it parses line by line interpreting words that has meaning in the JavaScript engine, and use them as instructions to tell a computer what to do. When this happens, it is actually storing stuff (e.g. functions, variables) in memory. This process of taking and running code line by line is known as a thread of execution.

Global execution context: The stuff (e.g. functions and variables) we declare or store in memory at the beginning of our code will be available anywhere inside of our application.

JavaScript thread: means that we go through our code line by line, from top to bottom, without skipping any line. This means that it is synchronous and single threaded, one thing at a time.

Functions: a way of wrapping up some instruction (code) to be done at a later stage, we can refer to this code later for execution. JavaScript never leaves a label without a value, and the default value is undefined. Anything we declare inside a function is committed to a little memory just for that function, not global memory. This is called function level or local.

Call Stack: When we call a function, we create an execution context and stack is a perfect way for tracking our execution context.
