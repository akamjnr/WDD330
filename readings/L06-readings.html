Forms - Chapter 8:

Forms: Made up of form element which contains form controls such as select menus, buttons and input fields which can be populated with information that is then processed once the form is submitted.

Form controls have an initial value specified in the HTML code that can be changed when a user enters information or interact with the form's interface. That value can also be changed dynamically using JavaScript.
Form objects have some useful properties and methods used to interact with the form.

The form.submit() method will submit the form automatically, and using this method won't trigger the form submit event.

The form.reset() method resets all the form controls back to their initial HTML specified values. A button with a type attribute of reset can also resets all controls without additional scripting. “Reset buttons are generally considered poor for usability, as they are too easy to click and then wipe out all the data that's been entered.”

The form.action property can be used to set the action attribute of a form, so it's sent to a different URL to be processed on the server.

Form Events: Focus event occurs when an element is focused on, e.g. an input element, this is when the cursor is placed inside the element (by either clicking or tapping on it or navigating to it using the keyboard).
The blur event occurs when user moves the focus away from the form element.
The change event occurs when a user moves the focus away from the form element after changing it. If a user clicks in an input field and makes no changes, then clicks elsewhere, the change event won't fire, but the blur event will.

Submitting a Form: Submit is possibly the most important form event that occurs when the form is submitted and this will send the content of the form to the server for processing. The form can be intercepted before it's sent using JavaScript by adding a submit event listener. The form can also be stopped from being submitted by using the preventDefault() method.

Form Controls: Common form control types:
▪ input fields, including text, passwords, check boxes, radio buttons, and file uploads.
▪ select menus for drop-down lists of options.
▪ textarea elements for longer text entry.
▪ button elements for submitting and resetting forms.

Form Validation: Process of checking whether a user has entered the information into a form correctly. Validation can occur, and is advisable to use it, on both the client and server side using JavaScript.
